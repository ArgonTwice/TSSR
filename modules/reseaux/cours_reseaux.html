1. Modèle OSI — 7 Couches

Le modèle OSI (Open Systems Interconnection) est un cadre conceptuel en 7 couches permettant de standardiser les communications réseau. Chaque couche a un rôle précis et communique avec les couches adjacentes.

#CoucheProtocoles / ExemplesRôle résumé
7ApplicationHTTP, HTTPS, FTP, DNS, SMTP, SSHInterface avec l'utilisateur final
6PrésentationTLS/SSL, JPEG, MPEG, ASCIIChiffrement, encodage, compression
5SessionNetBIOS, RPC, SMBOuverture/fermeture de sessions
4TransportTCP (fiable), UDP (rapide)Segmentation, contrôle de flux, ports
3RéseauIP, ICMP, ARP, OSPF, BGPAdressage IP et routage inter-réseaux
2LiaisonEthernet, Wi-Fi (802.11), VLANAdresses MAC, trames, accès au médium
1PhysiqueRJ45, fibre, ondes radioBits sur le support physique
Mnémotechniques :

7→1 : Alex Pète Sa Tête Rarement Le Premier

CritèreTCPUDP
ConnexionOrienté connexion (3-way handshake)Sans connexion
FiabilitéAccusé de réception, retransmissionAucune garantie

1.2 Établissement de connexion TCP (3-way handshake)

Client          Serveur
  |-- SYN -------->|   (Je veux me connecter, seq=100)
  |<-- SYN-ACK ----|   (OK, seq=200, ack=101)
  |-- ACK -------->|   (Reçu, ack=201)
  |== DATA =======>|   (Échange de données)

2. Adressage IPv4

Une adresse IPv4 est composée de 32 bits représentés en 4 octets décimaux (ex: 192.168.1.10). Elle comprend une partie réseau et une partie hôte, délimitées par le masque.

2.1 Classes d'adresses (historique)

Privées10.x / 172.16-31.x / 192.168.x—Usage interne (RFC 1918)
Loopback127.0.0.0/8—Test local (127.0.0.1)

3.2 DNS — Domain Name System

Le DNS traduit les noms de domaine en adresses IP. Il utilise une hiérarchie : racine → TLD (.fr, .com) → domaine → sous-domaine.

TypeRôleExemple
ANom → IPv4www.tssr.local → 192.168.1.10
AAAANom → IPv6www → 2001:db8::1
CNAMEAlias de nomftp → www.tssr.local
MXServeur mailtssr.local → mail.tssr.local
PTRIP → Nom (inverse)192.168.1.10 → srv.tssr.local
NSServeur de nomstssr.local → ns1.tssr.local

3.3 DHCP — Processus DORA

Client                              Serveur DHCP
  |-- DISCOVER (broadcast) -------->|   "Qui peut me donner une IP ?"
  |<-- OFFER (unicast/broadcast) ---|   "Je propose 192.168.1.50"
  |-- REQUEST (broadcast) --------->|   "J'accepte 192.168.1.50"
  |<-- ACK (unicast) ---------------|   "C'est confirmé, durée 8h"
